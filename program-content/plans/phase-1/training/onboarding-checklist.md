# Student Assistant Onboarding Checklist

*Copy this per student (e.g., `onboarding-<name>.md`) or track in the tracker. It's the trackable path from hire to certified Tier-B remediator. Gates are hard stops: a student doesn't work a tier live until the gate is passed.*

**Student:** __________  **Start date:** __________  **Supervisor:** __________

---

## Week 0 — Access & setup (local process docs cover the "how")
- [ ] Accounts: Canvas access, tracker access, Box/shared drive access
- [ ] Tools installed/licensed: Adobe Acrobat Pro, ABBYY FineReader, Equidox (if used), PAC 2024, NVDA
  - *See `../process-documentation/` tool-setup docs for exact steps.*
- [ ] Read `../README.md`, the `../remediation-procedures/00-decision-tree.md`, and the process-documentation README
- [ ] FERPA / student-data handling briefing (esp. before using any cloud tool like Equidox)
- [ ] Payroll/HR onboarding complete

## Stage 0 — Foundations
- [ ] WebAIM foundational articles (headings, alt text, tables, contrast)
- [ ] Section508 "Create Accessible Documents" intro
- [ ] One AccessibilityOnline / intro webinar
- [ ] **Concept check passed** (screen-reader basics, WCAG 2.1 AA overview, what "tagged" means) — supervisor sign-off: ______

## Stage 1 — Tier A training
- [ ] Microsoft Accessibility Checker practiced in Word, PowerPoint, Excel
- [ ] SOPs read & practiced: Word, PowerPoint, born-digital PDF, scanned PDF (OCR), images/alt text
- [ ] Core course (WebAIM Accessible Documents, if licensed) — certificate: ______
- [ ] Adobe Acrobat tutorials + Clemson PDF workflow completed
- [ ] Practiced on the **Tier A practice set** (documents with known issues)

### ✅ Tier A CERTIFICATION GATE
- [ ] Student remediates a fresh Tier A practice document unaided
- [ ] Passes **Acrobat Full Check** (0 fails) **and** (PDF) **PAC 2024** clean
- [ ] Passes **human QA:** reading order, heading logic, meaningful alt text, correct list/table structure
- [ ] Supervisor sign-off: __________  Date: ______
- [ ] **→ Cleared for live Tier A work.** First N live docs get 100% QA (see QA policy).

## Stage 2 — Tier B training
- [ ] Table remediation: Acrobat Table Editor (TH, scope, spans), complex tables
- [ ] Multi-column reading order; bookmarks for long docs
- [ ] PAC 2024 validation; **NVDA spot-check** technique (Say All, headings, Elements List, Tab)
- [ ] Tool proficiency: Equidox (volume) and/or ABBYY (tough OCR)
- [ ] Practiced on the **Tier B practice set**

### ✅ Tier B CERTIFICATION GATE
- [ ] Student remediates a fresh Tier B document (with a data table + figures) unaided
- [ ] Acrobat Full Check + PAC clean; NVDA spot-check sensible
- [ ] Human QA: table headers associate correctly, reading order correct, alt text meaningful
- [ ] Supervisor sign-off: __________  Date: ______
- [ ] **→ Cleared for live Tier B work.**

## Ongoing
- [ ] Knows the **Tier C escalation triggers** and how to hand off (`../remediation-procedures/stem-math-escalation.md`)
- [ ] Logs every item at intake and closes with production metrics (no exceptions)
- [ ] Writes a process note when they discover a reusable local trick or gotcha
- [ ] QA sampling rate stepped down as accuracy is proven

## Stage 3 — Growth (optional, for standouts)
- [ ] Deque University modules
- [ ] IAAP ADS credential path
- [ ] QA reviewer / peer mentor role

---
*Gates exist to protect students (from being handed work they aren't ready for) and disabled students (from bad remediation). When unsure whether someone is ready, don't sign the gate — give more practice.*
