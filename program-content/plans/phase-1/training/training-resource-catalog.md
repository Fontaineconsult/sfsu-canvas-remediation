# Training Resource Catalog — Document Remediation

*Reusable, verified external training for teaching student assistants document accessibility (PDF tagging, OCR, MS Office, alt text, WCAG 2.1 AA). We reuse existing curricula rather than building our own. URLs verified live 2026-07-06.*

> **⚠️ Access correction — read first.** SFSU is a **CSU** campus, **not** a California Community Colleges (CCC) campus. The CCC Accessibility Center catalog (PDF series, Accessibility Basics micro-courses, subsidized WebAIM cohorts) is **gated to CCC employees** and is **not** open to SFSU staff/students on its own. **Our equivalent free channel is the CSU Accessible Technology Initiative (ATI)**, which negotiates systemwide licenses. *Action item (Daniel): ask the SFSU ATI coordinator what's already licensed — ATI has historically covered WebAIM Accessible Document Training, CCC micro-course access for CSU, ABBYY FineReader, Equidox, UDOIT, PopeTech, and TidyUP. Confirm before paying for anything.* This supersedes the earlier plan note about "resolving CCC access."

## Recommended onboarding stack (cheapest-viable path)

1. **Free & authoritative first:** WebAIM articles → Section508 video modules → Microsoft in-app guides. Zero cost, no gatekeeping. Gets a student to competent-on-basics.
2. **Check CSU ATI (#5) before paying** — the WebAIM Accessible Documents course and CCC micro-courses may already be licensed free for SFSU.
3. **Core structured onboarding:** WebAIM Accessible Documents course (Office + PDF + Excel, certificate) — best single course for student workers.
4. **PDF tagging depth:** Adobe official tutorials + Clemson's free workflow.
5. **Tool training:** UDOIT/Cidi Labs docs for the Canvas scanning workflow; ABBYY and Equidox vendor guides for our licensed tools.
6. **Growth path for standouts:** Deque University → IAAP **ADS** certification.

See `onboarding-checklist.md` for how these map to our certification gates, and `README.md` for the curriculum sequence.
